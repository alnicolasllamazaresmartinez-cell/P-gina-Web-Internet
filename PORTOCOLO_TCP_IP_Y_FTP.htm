El protocolo TCP/IP es el conjunto de reglas que gobierna cómo se transmiten los datos a través de Internet. Su nombre proviene de los dos protocolos principales que lo componen:

·         TCP (Transmission Control Protocol): Se encarga de dividir los datos en paquetes pequeños y garantizar que lleguen correctamente a su destino, asegurándose de que se reensamblen en el orden correcto.

·         IP (Internet Protocol): Su función principal es dirigir los paquetes a través de la red, asignando una dirección IP única a cada dispositivo conectado. Este protocolo se encarga de la "entrega" de los paquetes a su destino correcto, basándose en las direcciones IP.

El FTP es un protocolo utilizado para transferir archivos entre un cliente y un servidor a través de Internet. Es muy utilizado para subir y bajar archivos de servidores web, así como para realizar copias de seguridad y compartir archivos grandes.

FTP funciona mediante el uso de dos canales:

1.      Canal de control: Se utiliza para enviar comandos entre el cliente y el servidor.

2.      Canal de datos: Se utiliza para la transferencia real de archivos.

A pesar de que FTP ha sido reemplazado en muchos casos por tecnologías más modernas, como SFTP (FTP seguro) y HTTP para la transferencia de archivos, sigue siendo ampliamente utilizado en servidores y aplicaciones especializadas.
